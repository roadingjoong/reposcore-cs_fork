# 오픈소스 기여 점수 산정 가이드

본 문서는 오픈소스 수업 프로젝트의 기여도(Pull Request 및 Issue)에 따른 점수 채점 기준을 안내합니다. 단순 오타 수정이나 이슈 제기만으로 점수를 독점하는 것을 방지하기 위해, **PR과 이슈의 비율에 따른 '인정 가능 최대 개수' 제한**이 적용됩니다.

---

## 1. 기본 배점 기준

### Pull Request (Merged PR)
- $P_{fb}$ : 기능 또는 버그 관련 PR (**3점**) ($P_{fb} = P_f + P_b$)
- $P_d$ : 문서 관련 PR (**2점**)
- $P_t$ : 오타 수정 PR (**1점**)

### Issue (Open / Resolved Issue)
- $I_{fb}$ : 기능 또는 버그 관련 이슈 (**2점**) ($I_{fb} = I_f + I_b$)
- $I_d$ : 문서 관련 이슈 (**1점**)

---

## 2. 점수 인정 가능 개수 제한 (Validity Limits)

무분별한 점수 획득을 방지하기 위해, 기여 내역은 다음 공식에 따라 유효한 개수만 산정됩니다.

**1) 점수 인정 가능 PR 개수 ($P_{\text{valid}}$)**  
기능/버그 PR($P_{fb}$)의 개수를 기준으로 문서 및 오타 PR의 인정 개수가 제한됩니다.
$$P_{\text{valid}} = P_{fb} + \min(P_d + P_t, 3 \times \max(P_{fb}, 1))$$
> *설명: 문서 및 오타 PR은 기능/버그 PR 개수의 최대 3배까지만 인정됩니다.  
> (단, 기능/버그 PR이 0개이더라도 문서/오타 PR은 최대 3개까지 기본 인정됩니다.)*

**2) 점수 인정 가능 이슈 개수 ($I_{\text{valid}}$)**  
이슈는 실제로 코드나 문서에 기여한 PR 개수($P_{\text{valid}}$)에 비례하여 제한됩니다.
$$I_{\text{valid}} = \min(I_{fb} + I_d, 4 \times P_{\text{valid}})$$
> *설명: 유효한 이슈 개수는 유효한 PR 개수의 최대 4배까지만 인정됩니다.*

---

## 3. 점수 최적화 산출 (Score Maximization)

학생의 점수를 최대로 보장하기 위해, 배점이 높은 기여(기능/버그)부터 우선적으로 유효 개수에 포함시킵니다.

**PR 최적화 계산**
1. $P_{fb}^* = \min(P_{fb}, P_{\text{valid}})$ (기능/버그 PR 최대 포함)
2. $P_d^* = \min(P_d, P_{\text{valid}} - P_{fb}^*)$ (남은 자리표에 문서 PR 포함)
3. $P_t^* = P_{\text{valid}} - P_{fb}^* - P_d^*$ (마지막 남은 자리에 오타 PR 포함)

**이슈 최적화 계산**
1. $I_{fb}^* = \min(I_{fb}, I_{\text{valid}})$ (기능/버그 이슈 최대 포함)
2. $I_d^* = I_{\text{valid}} - I_{fb}^*$ (남은 자리에 문서 이슈 포함)

---

## 4. 최종 점수 공식 (Final Score)

최적화된 기여 개수에 각 배점을 곱하여 최종 점수를 산출합니다.

$$S = 3P_{fb}^* + 2P_d^* + 1P_t^* + 2I_{fb}^* + 1I_d^*$$

## 5. 점수 산정 예시 (Scoring Examples)

복잡한 산정 공식을 쉽게 이해할 수 있도록, 세 가지 대표적인 학생의 기여 패턴을 예시로 정리했습니다.

### 예시 1 : 균형 잡힌 기여
기능 개발과 문서화, 이슈 제기를 적절히 병행한 학생입니다.

* **기여 내역:** 
  * PR: 기능/버그 2개, 문서 2개, 오타 1개
  * 이슈: 기능/버그 3개, 문서 2개
* **1단계 (유효 PR 계산):**
  * $P_{\text{valid}} = 2 + \min(2+1, 3 \times \max(2, 1)) = 2 + \min(3, 6) = 5$개 (모두 인정)
* **2단계 (유효 이슈 계산):**
  * $I_{\text{valid}} = \min(3+2, 4 \times 5) = \min(5, 20) = 5$개 (모두 인정)
* **3단계 (최종 점수):**
  * PR 점수: $(2 \times 3점) + (2 \times 2점) + (1 \times 1점) = 11점$
  * 이슈 점수: $(3 \times 2점) + (2 \times 1점) = 8점$
  * **최종 합계: 19점**

---

### 예시 2 : 오타/문서 PR 편중
기능 기여 없이 난이도가 낮은 문서나 오타 수정 PR만 대량으로 올린 학생입니다.

* **기여 내역:**
  * PR: 기능/버그 0개, 문서 5개, 오타 5개
  * 이슈: 문서 5개
* **1단계 (유효 PR 계산):**
  * $P_{\text{valid}} = 0 + \min(5+5, 3 \times \max(0, 1)) = 0 + \min(10, 3) = 3$개
  * *결과: 10개의 PR 중 최대 3개까지만 유효 PR로 인정됩니다.*
* **2단계 (PR 최적화):**
  * 인정받은 3자리를 배점이 높은 **문서 PR 3개**로 채웁니다.  
    (오타 PR 5개와 남은 문서 PR 2개는 버려짐)
* **3단계 (최종 점수):**
  * PR 점수: $(3 \times 2점) = 6점$
  * 이슈 점수: 유효 이슈 한도($4 \times 3 = 12$) 내이므로 문서 이슈 5개 인정 $(5 \times 1점) = 5점$
  * **최종 합계: 11점**

---

### 예시 3 : 이슈 편중
코드를 수정하는 PR은 거의 없이, 이슈만 과도하게 많이 오픈한 학생입니다.

* **기여 내역:**
  * PR: 기능/버그 1개, 문서 0개, 오타 0개
  * 이슈: 기능/버그 6개, 문서 4개 (총 10개)
* **1단계 (유효 PR 계산):**
  * $P_{\text{valid}} = 1 + \min(0, 3 \times 1) = 1$개
* **2단계 (유효 이슈 계산):**
  * $I_{\text{valid}} = \min(10, 4 \times 1) = \min(10, 4) = 4$개
  * *결과: 이슈를 10개나 올렸지만, PR 개수 제한에 걸려 4개만 인정됩니다.*
* **3단계 (이슈 최적화):**
  * 인정받은 4자리를 배점이 높은 **기능/버그 이슈 4개**로 채웁니다.
* **4단계 (최종 점수):**
  * PR 점수: $(1 \times 3점) = 3점$
  * 이슈 점수: $(4 \times 2점) = 8점$
  * **최종 합계: 11점**
